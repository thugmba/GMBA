# Example News Post

This is an example of how a news post looks when created as a GitHub Issue.

## What Administrators See (GitHub Issue):

**Title:**
```
Welcome to Tunghai Global MBA 2025
```

**Labels:** `news`

**Body:**
```markdown
## Summary
We are thrilled to welcome 50 new students from 15 countries to our 2025 Global MBA cohort.

## Content

![Welcome Event](https://via.placeholder.com/800x400/003366/ffffff?text=Welcome+Event+2025)
```

## What Visitors See (On Website):

The news would appear on:

1. **Homepage** (Latest News section) - Shows as a card with:
   - Featured image
   - Title
   - Date
   - Summary
   - "Read More" link

2. **News Archive Page** - Shows all news posts with:
   - Date badge
   - Title
   - Featured image
   - Summary
   - Link to full article

3. **Individual News Page** - Full article with:
   - Title
   - Date
   - Featured image
   - Complete content (formatted from markdown)
   - Back to news link

## Tips for Great News Posts

1. **Use descriptive titles** - Make it clear what the news is about
2. **Write a compelling summary** - This appears on the homepage
3. **Add images** - Visual content is more engaging
4. **Use headings** - Break up text for readability
5. **Include links** - Direct readers to relevant pages
6. **Proofread** - Check spelling and grammar
7. **Use markdown** - Format text, add lists, etc.

## Markdown Quick Reference

```markdown
## Heading 2
### Heading 3

**Bold text**
*Italic text*

- Bullet point
- Another point

1. Numbered item
2. Second item

[Link text](/GMBA/page)

![Image alt text](https://url-to-image.jpg)
```

## Important Notes

- The issue **must** have the `news` label to appear on the website
- The issue must be **open** (not closed) to be visible
- Images are automatically hosted by GitHub when you drag & drop
- Changes to the issue update the website immediately
- You can edit the issue anytime to update the news post
