# How to Post News - Using GitHub Issues

**Super Simple! No technical knowledge required!**

News posts are created as GitHub Issues with the "news" label. This method is even easier than editing files - just fill out a form!

## Quick Start (5 Steps)

1. Go to: https://github.com/thugmba/GMBA/issues/new/choose
2. Click **"Get started"** next to "News Post"
3. Fill in the title and content
4. Click **"Submit new issue"**
5. Done! Your news appears on the website immediately!

## Detailed Instructions

### Step 1: Go to Create New Issue

1. Visit: https://github.com/thugmba/GMBA/issues/new/choose
2. You'll see a "News Post" template
3. Click **"Get started"**

### Step 2: Fill in the Title

In the "Title" field at the top, write your news headline:
- Example: `New Scholarship Program for International Students`
- Keep it concise and descriptive

### Step 3: Write Your News Content

The template has sections to guide you:

#### Summary Section
```markdown
## Summary
Write a brief 1-2 sentence summary here.
This will appear in news listings on the homepage.
```

#### Main Content Section
```markdown
## Content

### Main Story
Write your full news article here.

You can use:
- **Bold text** for emphasis
- *Italic text* for quotes
- Bullet points like this
- ## Headings for sections
```

### Step 4: Add Images (Optional)

To add images to your news post:

1. **Upload the image:**
   - Drag and drop the image directly into the text box, OR
   - Click the image icon in the toolbar
   - GitHub will automatically upload it

2. **The markdown will appear automatically:**
   ```markdown
   ![image description](https://user-images.githubusercontent.com/.../image.jpg)
   ```

### Step 5: Submit

1. Make sure the **"news" label** is checked (it should be by default)
2. Click **"Submit new issue"** button
3. Your news is now live on the website!

## Formatting Guide

### Bold Text
```markdown
**This text will be bold**
```

### Italic Text
```markdown
*This text will be italic*
```

### Headings
```markdown
## Large Heading
### Medium Heading
#### Small Heading
```

### Bullet Points
```markdown
- First point
- Second point
- Third point
```

### Numbered Lists
```markdown
1. First item
2. Second item
3. Third item
```

### Links
```markdown
[Link text](https://example.com)
```

For internal links to your website pages:
```markdown

```

### Images
Drag and drop directly into the text box, or use:
```markdown

```

## Managing News Posts

### Edit a News Post

1. Go to: https://github.com/thugmba/GMBA/issues
2. Click on the news post you want to edit
3. Click the **"..."** menu (three dots) → **"Edit"**
4. Make your changes
5. Click **"Update comment"**

### Delete/Unpublish a News Post

**Option 1: Close the Issue (Hide from website)**
1. Open the news issue
2. Click **"Close issue"** button at the bottom
3. The news will no longer appear on the website

**Option 2: Remove the "news" label**
1. Open the news issue
2. Click the gear icon next to "Labels"
3. Uncheck "news"
4. The issue remains but won't show as news on the website

**Option 3: Delete completely**
1. Open the news issue
2. Scroll to the bottom
3. Click **"Delete issue"** (requires admin permissions)

### Re-publish a Closed News Post

1. Open the closed issue
2. Click **"Reopen issue"**
3. Make sure it has the "news" label
4. It will appear on the website again

## Adding Administrators

To give someone permission to post news:

1. Go to: https://github.com/thugmba/GMBA/settings/access
2. Click **"Invite a collaborator"**
3. Enter their GitHub username or email
4. Select **"Write"** permission
5. They'll receive an invitation email
6. Once they accept, they can post news!

## Troubleshooting

**Q: My news isn't showing on the website**
- Check that the issue has the "news" label
- Check that the issue is "Open" (not closed)
- Wait 1-2 minutes for the website to refresh
- Clear your browser cache

**Q: How do I add multiple images?**
- Just drag and drop multiple images into the text box
- They'll be uploaded automatically
- Arrange them in your content as needed

**Q: Can I schedule news for future posting?**
- Create the issue as a draft first
- When ready to publish, make sure it's open with the "news" label

**Q: Images aren't showing**
- Make sure you uploaded them via drag-and-drop or GitHub's image uploader
- Check that the image URL starts with `https://`
- Try uploading smaller images (< 10MB)

**Q: Can I use HTML?**
- Yes, but markdown is recommended for simplicity
- GitHub supports both markdown and HTML in issues

## Tips for Great News Posts

1. **Catchy titles** - Make titles clear and engaging
2. **Good summary** - The first 1-2 sentences are crucial
3. **Use images** - Photos make news more engaging
4. **Break it up** - Use headings and bullet points for readability
5. **Proofread** - Use the preview tab to check before submitting
6. **Add links** - Link to related pages on your website
7. **Be timely** - Post news while it's fresh!
